ПРИЛОЖЕНИЕ А. ПСЕВДОКОД АЛГОРИТМА АДАПТАЦИИ

Алгоритм выполняется каждые 5 мин по расписанию cron и обновляет выражения триггеров Zabbix через API trigger.update.

Алгоритм 1: AdaptiveThresholdUpdate

Вход: N – множество узлов; M – метрики; k, Δt, δ, α, β – параметры

Выход: обновлённые выражения триггеров Tij для всех ni ∈ N, mj ∈ M

1:  for each nᵢ ∈ N do
2:      dᵢ ← MedianRTT(nᵢ, last=10)          // скользящая медиана RTT
3:      for each mⱼ ∈ M do
4:          H ← GetHistory(nᵢ, mⱼ, [t−dᵢ−Δt; t−dᵢ])  // скорр. окно
5:          μᵢⱼ ← EWMA(H, α)                  // взвешенное среднее
6:          σᵢⱼ ← WeightedStd(H, α)           // взвешенное СКО
7:          (Lₜ, Bₜ) ← HoltUpdate(H, α, β)   // уровень и тренд
8:          X̂ᵢⱼ ← Lₜ + δ·Bₜ                  // прогноз на горизонт δ
9:          Tᵢⱼ ← μᵢⱼ + k·σᵢⱼ + X̂ᵢⱼ          // адаптивный порог
10:         expr ← "{host:metric.last()} > " + Tᵢⱼ
11:         ZabbixAPI.trigger_update(nᵢ, mⱼ, expr)
12:     end for
13: end for

Сложность: O(|N|·|M|·Δt) ≈ O(5040) при |N|=7, |M|=12, Δt=60

Реализация на Python 3 с использованием стандартных библиотек (statistics, requests) будет опубликована в открытом репозитории при принятии статьи к печати.
